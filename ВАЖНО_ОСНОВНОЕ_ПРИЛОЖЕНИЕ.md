# ⚠️ Важно: Основное приложение и веб-версия

## Проблема:

Основное приложение (`mobile/`) - это **React Native приложение**, которое:
- ✅ Работает на мобильных устройствах (Android/iOS)
- ✅ Может работать в веб-режиме через Expo для разработки
- ❌ **НЕ может быть собрано для статического хостинга** (GitHub Pages) автоматически

## Почему так:

React Native приложения используют нативные компоненты, которые:
- Требуют специальной сборки для веба
- Могут не работать в статическом окружении
- Нуждаются в сервере для работы

## Решения:

### Вариант 1: Использовать только AR игры (текущее решение)
- ✅ AR игры работают на GitHub Pages
- ✅ Доступны по HTTPS для камеры
- ❌ Нет остальных игр и функций

### Вариант 2: Задеплоить основное приложение отдельно
- Использовать **Expo** для веб-версии: `expo start --web`
- Или использовать **Vercel/Netlify** для React Native веб-версии
- Или создать отдельную веб-версию на чистом React

### Вариант 3: Запускать локально
- Основное приложение запускать локально через `expo start`
- AR игры доступны на GitHub Pages

## Рекомендация:

Для полного функционала лучше:
1. **AR игры** - на GitHub Pages (уже работает) ✅
2. **Основное приложение** - через Expo Go на телефоне или локально

Или создать упрощенную веб-версию основного приложения на React (не React Native).
