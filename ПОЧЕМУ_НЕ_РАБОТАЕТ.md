# Почему основное приложение не загружается на GitHub Pages

## Проблема:

**Основное приложение** (`mobile/`) - это **React Native приложение**, которое:
- ✅ Отлично работает на мобильных устройствах (Android/iOS)
- ✅ Может работать в веб-режиме для разработки (`expo start --web`)
- ❌ **НЕ может быть автоматически собрано** для статического хостинга (GitHub Pages)

## Почему так:

1. **React Native** использует нативные компоненты, которые требуют специальной обработки
2. **Expo export:web** может не работать в CI/CD окружении без интерактивного режима
3. **Статический хостинг** (GitHub Pages) не поддерживает серверные функции

## Что сейчас задеплоено:

✅ **AR игры** (`mobile/ar-games/`) - это обычное React приложение, работает отлично
❌ **Основное приложение** - не собралось, поэтому не задеплоилось

## Решения:

### Вариант 1: Использовать только AR игры (текущее)
- ✅ Работает на GitHub Pages
- ✅ HTTPS для камеры
- ❌ Нет остальных игр и функций

### Вариант 2: Запускать основное приложение локально
```bash
cd mobile
npm start
# Затем откройте в браузере или используйте Expo Go на телефоне
```

### Вариант 3: Создать упрощенную веб-версию
Создать отдельную веб-версию основного приложения на чистом React (не React Native)

### Вариант 4: Использовать другой хостинг
- **Vercel** - поддерживает React Native веб-версии лучше
- **Netlify** - также может работать
- **Expo** - для мобильных приложений

## Рекомендация:

Для полного функционала лучше всего:
1. **AR игры** - на GitHub Pages (уже работает) ✅
2. **Основное приложение** - через Expo Go на телефоне или локально

Или я могу создать упрощенную веб-версию основного приложения на React.
